# اپلیکیشن استعدادیابی ورزشی (تنیس)

## مقدمه

این پروژه با هدف طراحی و پیاده‌سازی یک اپلیکیشن استعدادیابی ورزشی در رشته تنیس انجام شده است. تمرکز اصلی پروژه بر استفاده از **مدل‌های ماشین لرنینگ** و **پردازش تصویر و ویدیو** برای تحلیل ویژگی‌های فیزیکی و حرکتی افراد و ارائه یک پیشنهاد اولیه درباره مناسب بودن آن‌ها برای رشته تنیس است.

اپلیکیشن شامل دو بخش اصلی است:

1. بخش متنی (داده‌های عددی کاربر)
2. بخش ویدیویی (تحلیل حرکت فورهند با Pose Detection)

---

## ساختار کلی پروژه

پروژه از نظر منطقی به سه بخش تقسیم می‌شود:

- جمع‌آوری ورودی‌ها (متنی و ویدیویی)
- استخراج ویژگی‌ها
- پیش‌بینی با استفاده از مدل‌های ماشین لرنینگ

رابط کاربری با استفاده از کتابخانه **Gradio** پیاده‌سازی شده است تا کاربر بتواند به‌صورت ساده اطلاعات خود را وارد کرده و ویدیو بارگذاری کند.

---

## بخش متنی (Text-based Analysis)

### ورودی‌ها

در بخش متنی، اطلاعات زیر از کاربر دریافت می‌شود:

- سن (عدد گسسته)
- قد (سانتی‌متر)
- طول دست‌ها (سانتی‌متر)
- زمان تست چابکی
- زمان عکس‌العمل
- جنسیت

### مدل ماشین لرنینگ مورد استفاده

برای این بخش از مدل **Random Forest Classifier** استفاده شده است.

دلایل انتخاب Random Forest:

- عملکرد مناسب روی داده‌های عددی
- مقاومت در برابر نویز
- عدم نیاز به نرمال‌سازی پیچیده داده‌ها



مدل با داده‌های عددی آموزش داده شده و خروجی آن یک برچسب باینری است:

- 1 : مناسب برای تنیس
- 0 : پیشنهاد سایر رشته‌ها

---

## بخش ویدیویی (Video-based Analysis)

### ورودی

در این بخش کاربر یک ویدیوی کوتاه از حرکت **فورهند تنیس** بارگذاری می‌کند.

### استخراج ویژگی (Pose Detection)

برای استخراج ویژگی‌های حرکتی از کتابخانه **MediaPipe Pose** استفاده شده است. این کتابخانه موقعیت مفاصل اصلی بدن (شانه، آرنج، مچ، لگن و زانو) را در هر فریم استخراج می‌کند.

ویژگی‌های استخراج‌شده شامل:

- زاویه آرنج دست غالب
- تغییرات زاویه شانه در طول حرکت
- پایداری بالاتنه
- هماهنگی حرکت دست و تنه

برای هر ویدیو، میانگین و واریانس این ویژگی‌ها محاسبه شده و به‌عنوان بردار ویژگی نهایی استفاده می‌شود.

### مدل ماشین لرنینگ مورد استفاده

برای تحلیل ویدیو از یکی از مدل‌های زیر استفاده شده است:

- Random Forest Classifier
  یا
- Support Vector Machine (SVM)

این مدل‌ها روی بردار ویژگی استخراج‌شده از Pose Detection آموزش داده شده‌اند.

## ترکیب نتایج (Fusion)

خروجی نهایی اپلیکیشن با ترکیب نتایج دو بخش به‌دست می‌آید:

- پیش‌بینی مدل متنی
- پیش‌بینی مدل ویدیویی

در ساده‌ترین حالت، اگر هر دو مدل نتیجه مثبت بدهند، فرد به‌عنوان «دارای استعداد اولیه برای تنیس» معرفی می‌شود.

---

## رابط کاربری (UI)

رابط کاربری با **Gradio** طراحی شده است و شامل:

- اسلایدرها و فیلدهای عددی برای داده‌های متنی
- دکمه بارگذاری ویدیو
- نمایش نتیجه نهایی به‌صورت متنی

سادگی رابط کاربری باعث افزایش سرعت استفاده و کاهش خطای کاربر می‌شود.
